# Phase 1: System Architecture and Project Setup

## What Was Built

- Full monorepo folder structure for `mobile/`, `backend/` (7 microservices +
  `shared/`), `database/`, `docker/`, `docs/`.
- `shared/schemas/`: Pydantic contracts every service will use -
  `AudioChunk`, `TranscriptEvent`, `TranslationEvent`, `SynthesizedAudioEvent`,
  `CallSession`, plus shared enums (`LanguageCode`, `CallStatus`,
  `ProcessingStage`).
- `shared/messaging/`: `MessageBroker` abstract interface +
  `RedisStreamBroker` concrete implementation, so services depend on an
  abstraction rather than directly on Redis (Dependency Inversion).
- `shared/logging/`: structured JSON logging config, shared by every service.
- `database/schema.sql`: initial tables - `users`, `calls`,
  `call_participants`, `conversation_turns`.
- Health-check-only FastAPI skeleton (`main.py`, `Dockerfile`,
  `requirements.txt`) for all 7 backend services.
- `docker/docker-compose.yml` wiring Postgres, Redis, and all services
  together with health-check-gated startup ordering.
- `.env.example`, `.gitignore`, root `README.md`.

## Key Decisions

| Decision | Choice | Reasoning |
|---|---|---|
| Communication style | Microservices | Independent scaling/failure isolation per pipeline stage |
| Inter-service data flow | Redis Streams | Persistent, consumer-group support, simpler than gRPC to start |
| STT / Translation / TTS | Cloud APIs first, behind provider interfaces | Fastest path to a working prototype; swappable later |
| Primary keys | UUID (not serial int) | IDs can be generated by services before a DB row exists (e.g. call_id used in Redis before Postgres insert) |
| Auth model (planned) | Phone number + OTP (India-first) | Matches common India app UX; password is optional/nullable in schema |

## Open Items for Later Phases

- Alembic migration setup (`database/migrations/`) - currently only a raw
  `schema.sql` bootstrap; Phase 4 or a dedicated DB-tooling phase should
  introduce versioned migrations.
- Nginx / TURN server config (`docker/nginx/`) - needed for Phase 5 (WebRTC),
  left empty for now.
- CI pipeline - not yet in scope per the user's phase ordering, but noted as
  a natural fit for Phase 16 (Docker Deployment) or Phase 17 (Cloud
  Deployment).

## How to Verify This Phase

```bash
cp .env.example .env
docker compose -f docker/docker-compose.yml --env-file .env up --build
# In another terminal:
for port in 8000 8001 8002 8003 8004 8005 8006; do curl -s http://localhost:$port/health; echo; done
```

All 7 services should return a JSON health payload, and Postgres should have
the 4 tables from `schema.sql` created automatically on first boot.
